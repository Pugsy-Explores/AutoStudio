# `agent_v2/schemas/` — Pydantic data models (no logic)

---

## 1. Purpose

**Does:** Define strict, typed data structures for the AgentV2 system. All models are Pydantic v2 BaseModels with validation. No runtime logic, no tool calls, no orchestration.

**Does not:** Execute any business logic. Schemas are pure structure and validation.

---

## 2. Responsibilities

```text
✔ owns
  PlanDocument, PlanStep, PlanMetadata
  ExecutionResult, ExecutionTask, TaskStatus
  ExplorationResult, FinalExplorationSchema
  ReplanContext, ReplanRequest, ReplanResult
  PlannerDecision, PlannerPlanContext
  AgentState
  Trace, TraceStep, TraceMetadata
  ToolCall, ToolResult, ToolError
  ExecutionPolicy, FailurePolicy

❌ does not own
  Runtime execution loops
  Dispatcher logic
  Retrieval or editing implementations
```

---

## 3. Key schemas by domain

### Planning

| Schema | Role |
|--------|------|
| `PlanDocument` | Structured plan with steps, engine hints, risk, metadata |
| `PlanStep` | Individual step (action, args, dependencies) |
| `PlannerDecision` | High-level decision (explore, act, plan, replan, stop, synthesize) |
| `PlannerPlanContext` | Input to PlannerV2 (exploration, session, conversation, memory fields) |
| `PlanState` | Runtime plan state (cursor, completed, pending steps) |

### Execution

| Schema | Role |
|--------|------|
| `ExecutionResult` | Result of executing a plan step |
| `ExecutionTask` | DAG task node (tool, args, dependencies, status) |
| `TaskScheduler` | DAG execution schedule (tasks, edges) |
| `TaskStatus` | Task state (pending, running, completed, failed) |
| `ExecutionPolicy` | Retry limits, budget caps, safety rules |
| `FailurePolicy` | Recovery rules on step failure |

### Exploration

| Schema | Role |
|--------|------|
| `ExplorationResult` | Raw exploration output |
| `FinalExplorationSchema` | Bundled exploration summary, confidence, evidence |
| `ExplorationCandidate` | Candidate file or symbol for inspection |
| `ExplorationItem` | Single exploration evidence item |
| `QueryIntent` | Structured query intent from QIP |

### Replan

| Schema | Role |
|--------|------|
| `ReplanContext` | Context for replanning (original plan, failures) |
| `ReplanRequest` | Request to replan |
| `ReplanResult` | Result of replan (new plan, reasoning) |

### Memory and state

| Schema | Role |
|--------|------|
| `AgentState` | Top-level state (context, metadata, instruction, exploration_result) |
| `SessionMemory` | Planner session context (active file, last tool) |

### Observability

| Schema | Role |
|--------|------|
| `Trace` | Execution trace |
| `TraceStep` | Single trace step (tool, inputs, outputs, timing) |
| `TraceMetadata` | Trace metadata |

### Tool

| Schema | Role |
|--------|------|
| `ToolCall` | Tool invocation specification |
| `ToolResult` | Tool output |
| `ToolError` | Tool error details |

---

## 4. Data flow

Schemas flow through the runtime as immutable DTOs:

```
runtime → schemas → validation → runtime
```

Example: `PlanDocument` is generated by `PlannerV2`, validated by `PlanValidator`, then consumed by `PlanExecutor` to build `ExecutionTask` instances.

---

## 5. Current design decisions

- **Pure Pydantic:** All models use Pydantic v2 for validation.
- **No circular dependencies:** Import order respects DAG (see `__init__.py`).
- **Separate from logic:** No business logic in schema files.
- **Immutable where possible:** Schema instances should be treated as data transfer objects.

---

## 6. What it DOES NOT do

- Does not execute tools.
- Does not orchestrate workflows.
- Does not implement pluggers or routers.
- Does not persist to disk (that's handled by runtime layers).

---

## 7. Integration points

- **Upstream:** Runtime modules create and populate schemas.
- **Downstream:** Runtime modules read and validate schemas.
- **Tests:** Schema fixtures used in test suites.
